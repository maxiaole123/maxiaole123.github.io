# 隐私政策撰写规范与常见违规点


马晓乐 | 东润律师事务所 法律从业者

隐私政策是APP合规的第一道门槛。据监管通报，约32%的违规案例与隐私政策相关。本文梳理撰写规范与高频违规点。


法定内容要求


根据PIPL第17条及《App违法违规收集使用个人信息行为认定方法》，隐私政策至少应包含：


1. 个人信息处理者的名称和联系方式

2. 个人信息的处理目的、处理方式

3. 处理的个人信息种类、保存期限

4. 个人行使权利的方式和程序（查阅、更正、删除、撤回同意、注销账户、获取副本）

5. 涉及敏感个人信息的，应特别告知处理的必要性及对个人的影响

6. 涉及向第三方提供的，应告知接收方信息


撰写五原则


**原则一：独立成文。** 隐私政策应为独立文件，不应作为用户协议的一部分。


**原则二：语言通俗。** 避免法律术语堆砌，使用普通用户能理解的语言。推荐采用"我们收集什么→用来做什么→和谁共享→如何保护→你的权利"的叙事结构。


**原则三：分层告知。** 首次打开APP时弹窗展示核心条款摘要，完整隐私政策通过链接查看。摘要应覆盖"收集什么+用来干什么"两个最核心问题。


**原则四：及时更新。** 处理目的、方式、范围发生变化时，应及时更新隐私政策并重新取得同意。


**原则五：如实描述。** 隐私政策描述必须与实际行为一致。写的和做的不一样，是监管处罚的重灾区。


常见违规点TOP5


1. 首次运行不弹窗提示隐私政策

2. 隐私政策中缺漏个人信息种类（实际收集了但政策未列出）

3. 同意前即开始收集个人信息

4. 未单独列出敏感个人信息的处理规则

5. 注销账户路径描述不清或实际无法操作
